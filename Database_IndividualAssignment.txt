Discussion:
A materialized view stores the result of the query, so it must be updated whenever the Part table changes. If a new row is inserted with a partCost greater than 1000, the partNo should be added to the view. If a part's cost is updated, the view should also be updated by adding the partNo if the cost becomes greater than 1000, or removing it if the part no longer has any contract with a cost above 1000. Similarly, when a row is deleted, the partNo should only be removed from the view if there are no other contracts for that part with a cost greater than 1000.

The view can be maintained without accessing the Part table only when it is certain that the change cannot affect other rows. For example, if a new row with partCost greater than 1000 is inserted, the partNo can simply be added to the view. However, if a row with partCost greater than 1000 is updated or deleted, the Part table must be checked to determine whether another contract for the same partNo still has a cost above 1000 before removing it from the view.
